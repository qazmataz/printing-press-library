# Gravitus CLI — Absorb Manifest

## Sources Evaluated
- **hevycli** (Go CLI, github.com/obay/hevycli) — strongest direct competitor; Hevy-specific
- **hevy-mcp** (Node MCP, github.com/chrisdoc/hevy-mcp) — MCP server for Hevy
- **HevyWorkoutAnalyzer** (Python, github.com/CyberGiant7/HevyWorkoutAnalyzer) — analytics layer
- **LiftShift** (github.com/aree6/LiftShift) — multi-app analytics (Hevy, Strong, Lyfta); plateau detection, muscle heatmaps
- **Strong app CSV ecosystem** — CSV export + analytics
- **fit-cli** (github.com/DevLifts/fit-cli) — simple fitness CLI with 1RM calc
- **No Gravitus CLI exists** — we are first and only

## Constraints
Gravitus exposes no JSON API. All data comes from authenticated HTML pages:
- `GET /users/{user_id}/` — workout list (paginated, 10/page)
- `GET /workouts/{workout_id}/` — workout detail
- `GET /exercises/{exercise_slug}/` — exercise history
- Write operations (create/update workout) are NOT accessible via web scraping

## Transcendence Features (only possible with our approach)

| # | Feature | Command | Buildability | Why Only We Can Do This |
|---|---------|---------|--------------|------------------------|
| 1 | Data export | `export` | hand-code | Gravitus has zero export feature. We are the only way to get your own data out. |
| 2 | PR timeline | `exercises prs --timeline` | hand-code | Requires joining all workouts × sets × dates locally. No single API call exists. |
| 3 | Plateau detection | `exercises plateau --weeks 4` | hand-code | Requires comparing 1RM estimates across time windows — only possible with full local history. |
| 4 | Volume trend heatmap | `stats volume --by-week` | hand-code | Weekly volume per muscle group over time — requires aggregating across all synced workouts. |
| 5 | Muscle balance score | `stats balance` | hand-code | Push vs pull volume ratio, bilateral balance — only computable with full local history. |
| 6 | Workout frequency gap | `stats gaps` | hand-code | Detects days with no training, longest gaps, rest patterns — cross-session aggregation. |
| 7 | Session comparison | `workouts compare <id1> <id2>` | hand-code | Compares two workouts side-by-side: volume delta, weight changes, set counts. |
| 8 | Training calendar | `stats calendar [year]` — GitHub-style dot grid | hand-code | Requires full date index from local store; no single API endpoint provides this. |
| 9 | AI agent context | `agent-context` + `--agent` flag on all commands | spec-emits | Structured JSON output for LLM consumption — agents can query your training history. |
